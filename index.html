Holon is a hybrid workforce platform where every person runs a lightweight local AI team, and those teams connect through cloud‑managed proxy employees into a larger human–AI network.

The Holon Principle

Each unit is a whole — and a part.

A holon is something that is both a complete whole and a node inside a larger whole. Holon applies that idea to work: every local app is a complete team desk, and at the same time a node in a larger network of human and AI teams.

Complexity grows through connections between teams — not by piling more agents inside one app.

Hybrid Workforce Model

Not chatbots. Not autonomous swarms.

The durable workplace is a hybrid: real people, local AI teams, proxy employees, cloud coordination, and auditable handoffs. Holon makes that hybrid first‑class instead of bolting it onto a single agent UI. 

# A node = one person's local Holon desk
Owner
  └─ local AI staff          execute bounded assignments
  └─ proxy staff             stand in for a remote person / node
  └─ inbound missions        work other nodes sent to you
  └─ outbound assignments    work you sent to others
  └─ deliverables            durable returned artifacts
Core Concepts

Six pieces — nothing more.

The vocabulary is intentionally small. The owner manages outcomes, not agent graphs.

Node

The local desk

One real person's app: roster, inbox, outbound work, deliverables, policies, connections. Run local or hosted.

Local AI Staff

Bounded executors

Agents that run through a runtime adapter (Hermes, OpenAI Agents SDK, browser workers, …) and return events or files.

Proxy Staff

People as first‑class staff

A local identity that represents a remote person or remote Holon node. Assign work the same way you assign to AI.

Mission

Inbound work

What another node sent you. Accept, reject, ask, delegate, or submit a deliverable — with a clear owner and state.

Assignment

Outbound work you own

Routed to local AI, proxy staff, or yourself — with routing metadata so you can always see where the work lives.

Deliverable

The durable artifact

Not just a chat message. A titled, attributed result attached back to the assignment that produced it.

Core Protocol

Dispatch → Accept → Execute → Return.

The minimum protocol is a mission going out to another node and a deliverable coming back — with handoffs that never fail silently.

1Owner assigns to proxy staff 2Mission sent to remote node 3Remote inbox: accept · reject · forward 4Execute locally or delegate 5Submit deliverable 6Attach back to original assignment

Holon does

  • Connect human‑owned local teams
  • Make proxy staff visibly tied to real people
  • Give every handoff a state and a returned artifact
  • Treat the cloud as a connector + reliability layer
  • Keep accountability human

Holon avoids

  • Deep agent hierarchies inside one workspace
  • Generic workflow / drag‑drop builders
  • Black‑box swarms that own all work
  • Silent failure on remote handoffs
  • Agent graph editors as the main UX
